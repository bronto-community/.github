# Contributing to Bronto Community projects

Thanks for your interest in contributing! 🦕

Projects in the [bronto-community](https://github.com/bronto-community) org are community-maintained by the Bronto team and friends. Contributions of all kinds are welcome — code, docs, ideas, and bug reports.

## Ways to contribute

- **Report a bug or request a feature** — open an issue on the relevant repository. Include enough context to reproduce or understand it.
- **Submit a pull request** — fork the repo, make your change, and open a PR. Keep PRs focused; small changes get reviewed faster.
- **Propose a new project** — built something that fits the community org (tutorials, agents, observability tooling, Bronto-related experiments)? Open an issue on any repo describing it and we'll figure out the best home for it.

## Ground rules

- Be kind and constructive — we're all here to build useful things together.
- Check a project's own README and CONTRIBUTING file first; per-repo guidelines take precedence over this one.
- Don't include secrets, customer data, or internal-only material in issues, code, or commit history.

## A note on support

These projects are **not officially supported by Bronto** — no SLAs, best-effort maintenance. For official integrations and tooling, see [github.com/brontoio](https://github.com/brontoio).
